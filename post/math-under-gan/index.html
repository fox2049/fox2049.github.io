GAN 的数学表示与推导

2021-04-14 11 min read # gan

GAN 的基本原理有各种通俗易懂的解释，可是其数学推导因为涉及到微积分、概率论和信息论的基础知识，理解起来相对难一些。当然，这也是 GAN 的精妙所在。GAN 数学推导的目的就是为了证明其 Loss 函数最低时，生成样本的分布与真实样本的分布相同。与此同时，判别器对生成样本判断为真的概率为0.5，即判别器已经无法找到生成样本和真实样本的明显区别，只能对其真假随机进行判断。

符号定义

PdataP_{data}Pdata​：真实数据的分布。

zzz : 简单的随机分布。例如高斯分布或者均匀分布，有时也被称之为随机噪音。

PzP_zPz​：zzz的概率分布。

xxx：生成样本。这个样本是 zzz 通过生成器 GGG 后的输出，即 x=G(z)x=G(z)x=G(z) ,后面为了简化表示也可表示为GGG。

D(x)D(x)D(x)：xxx是真实样本的概率，假定真实样本未经过神经网络，判定器可以直接比较真实样本和生成样本的区别。后面为了简化表示也可表示为DDD。

PgP_{g}Pg​：生成的数据的概率分布。

E\mathbb{E}E：数学期望，试验中每次可能结果的概率乘以其结果的总和，它反映随机变量平均取值的大小。连续型随机变量的期望也可以表示成积分的形式：

E(x)=∫xf(x)dx\mathbb{E}(x) = \int xf(x) dx E(x)=∫xf(x)dx

DKLD_{KL}DKL​：KL散度，又称相对熵，信息散度，信息增益。KL 散度是两个概率分布 P 和 Q 差别的非对称性的度量。在经典境况下，P 表示数据的真实分布，Q 表示数据的理论分布，模型分布。

DKL(P∥Q)=∑k=1nPilog(PiQi)D_{KL} (P \parallel Q) = \sum _{k=1}^n P_i log(\frac{P_i}{Q_i}) DKL​(P∥Q)=k=1∑n​Pi​log(Qi​Pi​​)

DJSD_{JS}DJS​：JS 散度，度量两个概率分布的相似度，是基于 KL 散度的变体，解决了 KL 散度非对称的问题。KL 散度和 JS 散度度量的时候都有一个问题：如果两个分布 P,Q 距离较远，完全没有重叠的时候，KL 散度是没有意义的，在学习的时候，这就意味着在这一点的梯度为 0，即梯度消失了。

DJS(P∥Q)=12DKL(P∥P+Q2)+12DKL(Q∥P+Q2)D_{JS} (P \parallel Q) = \frac{1}{2}D_{KL} (P \parallel {\frac{P+Q}{2}}) + \frac{1}{2}D_{KL} (Q \parallel {\frac{P+Q}{2}}) DJS​(P∥Q)=21​DKL​(P∥2P+Q​)+21​DKL​(Q∥2P+Q​)

数学表示

判别器

判别器的功能是对所生成的样本xxx进行概率评判，真为1，假为0。分为两种情况：

  1. 当判定器认为 xxx 来自于 PdataP_{data}Pdata​ 时，D(x)D(x)D(x) 的值就需要向1增大↑\uparrow↑。
  2. 当判定器认为 xxx 来自于PgP_gPg​,即判定出zzz来自于 PzP_zPz​，D(x)D(x)D(x) 即 D(G(z))D(G(z))D(G(z)) 的值就需要向0减小↓\downarrow↓。

为了便于后续计算，将 D(x)D(x)D(x) 前面统一加上 logloglog 表示(本文 logloglog 指自然对数)。综合考虑这两种情况，因为 D(x)D(x)D(x) 是位于[0,1]的值， D(G(z))D(G(z))D(G(z)) 变为1−D(G(z))1 - D(G(z))1−D(G(z)) 时，其变化就和情况1中的 D(x)D(x)D(x) 保持一致。为了使判别器达到更好的效果，就要使两方的平均值达到最大。平均值可以用数学期望 E\mathbb{E}E 来表示。因此判别器的目标函数可以写为：

max⁡D=Ex∼Pdatalog[D(x)]+Ez∼Pzlog[1−D(G(z))]\max _D = \mathbb{E}_{x \sim P_{data}}log[D(x)] + \mathbb{E}_{z \sim P_{z}}log[1- D(G(z))] Dmax​=Ex∼Pdata​​log[D(x)]+Ez∼Pz​​log[1−D(G(z))]

目标是为了找到一个D，使上式达到最大值。

生成器

对于生成器来说，它的训练目标是为了让判别器的能力降低，与判别器相反，当zzz来自于 PzP_zPz​时，需要使得 D(G(z))D(G(z))D(G(z)) 向1增大↑\uparrow↑，即 log[1−D(G(z))]log[1- D(G(z))]log[1−D(G(z))] 向0减小↓\downarrow↓。因为它没有从真实分布中取得输入，能改变的只有判别器目标函数的后半部分，其目标函数为：

min⁡G=Ez∼Pzlog[1−D(G(z))]\min _G = \mathbb{E}_{z \sim P_{z}}log[1- D(G(z))] Gmin​=Ez∼Pz​​log[1−D(G(z))]

目的是为了找到一个G，使上式达到最小值。

目标函数

根据判别器和生成器的目标函数，可以将 GAN 的总体目标函数可以写成：

min⁡Gmax⁡D=Ex∼Pdatalog[D(x)]+Ez∼Pzlog[1−D(G(z))]\min _G \max _D = \mathbb{E}_{x \sim P_{data}}log[D(x)] + \mathbb{E}_{z \sim P_{z}}log[1- D(G(z))] Gmin​Dmax​=Ex∼Pdata​​log[D(x)]+Ez∼Pz​​log[1−D(G(z))]

这里有两个概念：Explicit Density Model 与 Implicit Density Mode ,区别为是否需要计算出一个明确概率分布的密度函数，在 GAN 中，其目的在于生成数据，对于其生成的数据分布密度函数是什么样并不关心，因此属于 Implicit Density Mode。
