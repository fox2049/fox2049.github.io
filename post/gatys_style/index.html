Gatys 风格迁移详解

2021-04-18 6 min read # style

Gatys 等人发表的 'Image Style Transfer Using Convolutional Neural Networks' 使用了 CNN 来进行图像风格迁移，引起了风格迁移研究的浪潮，后面也相继产生了迁移效果和时间都更佳的方式。不过，这篇论文仍是入门风格迁移不可忽略的存在。

Gatys 所用的方法为图像生成式的风格迁移方法，就是通过将一张噪音图片（可以理解成老式电视机的雪花画面）与两张图片进行对比，这两张图片分别为风格的来源（风格图）、内容的来源（内容图），根据对比的结果进行更新迭代。随机噪音图从而逐渐变化为一张“风格迁移”的图片，即包含风格图的风格又包含内容图的内容。这时就有有两个问题：

  1. 为什么这样进行对比？
  2. 如何量化图片间的差异？

WHY

上图直观的表达出了图像在不同层的表现。Gatys 用了比较成熟的 VGG-19 作为 CNN，这个 CNN 有个非常奇妙的特性：

  • 内容层面：网络的底层特征更多的表现输入图像的像素信息，而高层特征更多的表现图像的物体和布局信息，通俗的来讲就是仅能够根据形状和位置判断图像信息，纹理和颜色信息丢失的比较严重。
  • 风格层面：网络的不同层表现出不同的风格特征，具有不同的风格特点。当然，这里 Gatys 做了一个假设，图片的风格就是图像的纹理和颜色特征。至于图片的风格究竟是什么，这就是另外一个话题了。

根据以上两个特性，噪音图像就有了可以对比和参考的对象。内容上选取一个高层的输出与“噪音图”进行对比，风格上使用多层的特征进行对比，最后加权确定图片的差异。

HOW

上图是论文中最为重要的一张图，通过这张图可以很清楚的看到图片迁移的具体步骤。其中 a⃗\vec{a}a 为风格来源图，p⃗\vec{p}p​ 为内容来源图，x⃗\vec{x}x 为噪音图片也是最终要生成的“风格迁移”图片。分别通过 VGG-19 网络。

内容差异

内容差异相对比较简单，只需要关注选出的那一层 lll ，将 p⃗\vec{p}p​ 在这一层的 feature maps 与 x⃗\vec{x}x 产生的做对比，就是将不同 feature map 的不同位置做方差即可。

Lcontent(p⃗,x⃗)=∑i,j(Fij−Pij)2\mathcal{L}_{content}(\vec{p}, \vec{x}) = \sum_{i, j}(F_{ij} - P_{ij})^2 Lcontent​(p​,x)=i,j∑​(Fij​−Pij​)2

其中 FijF_{ij}Fij​ 代表在选定的那一层网络上，第 iii 个feature map 上第 jjj 个数值.

风格差异

风格差异会涉及到 Gram 矩阵对风格的表示，至于Gram 矩阵为何能够表示出图片的“风格”，这就是另一个复杂的问题了，涉及到再生核希尔伯特空间等[1]。这里，我们假定 Gram 矩阵可以表示出风格特征。那么对于生成图片来说，第 lll 层 CNN 中任意两个feature map，例如第 iii 个与 第 jjj 个所组成的Gram矩阵为:

Gijl=∑kFiklFjklG_{ij}^l = \sum_{k}F_{ik}^l F_{jk}^l Gijl​=k∑​Fikl​Fjkl​

其中 kkk 为 feature map 上第 kkk 个数值。要注意，这个公式中的 jjj 代表的含义与 Lcontent\mathcal{L}_{content}Lcontent​ 的表示有所不同。
将生成图的 Gram 矩阵与风格图的 Gram 矩阵的第 lll 层网络做方差即可比较出第 lll 层的风格差异 ElE_lEl​。

El=∑i,j(Gijl−Aijl)2E_l = \sum_{i, j}(G_{ij}^l - A_{ij}^l)^2 El​=i,j∑​(Gijl​−Aijl​)2

其中 AijlA_{ij}^lAijl​ 为风格图第 lll 层 CNN中第 iii 个与 第 jjj 个feature map所组成的Gram矩阵。因为风格差异要考虑每层的不同的风格，因此对每一层的误差进行加权后得到总的风格差异为：

Lstyle(a⃗,x⃗)=∑l=0LwlEl\mathcal{L}_{style}(\vec{a}, \vec{x}) = \sum_{l=0}^L w_l E_l Lstyle​(a,x)=l=0∑L​wl​El​

其中的 wlw_lwl​ 为每一层的风格差异权重。

总体差异

为了实现归一化[2]，Gatys 在论文中的损失函数分别增加了系数。

Lcontent(p⃗,x⃗,l)=12∑i,j(Fijl−Pijl)2\mathcal{L}_{content}(\vec{p}, \vec{x}, l) = \frac{1}{2}\sum_{i, j}(F_{ij}^l - P_{ij}^l)^2 Lcontent​(p​,x,l)=21​i,j∑​(Fijl​−Pijl​)2

El=14N2M2∑i,j(Gijl−Aijl)2E_l = \frac{1}{4N^2M^2}\sum_{i, j}(G_{ij}^l - A_{ij}^l)^2 El​=4N2M21​i,j∑​(Gijl​−Aijl​)2

其中的 NNN 为本层中feature map的个数，MMM 为feature map的容积，对于 feature map 来说就是 H×WH \times WH×W。不过这个并不是很重要，因为最终的总体差异 Ltotal\mathcal{L}_{total}Ltotal​ 中仍会对 Lcontent\mathcal{L}_{content}Lcontent​ 与 Lstyle\mathcal{L}_{style}Lstyle​ 增加系数。

Ltotal(p⃗,a⃗,x⃗)=αLcontent(p⃗,x⃗)+βLstyle(a⃗,x⃗)\mathcal{L}_{total}(\vec{p},\vec{a}, \vec{x}) = \alpha\mathcal{L}_{content}(\vec{p}, \vec{x}) + \beta\mathcal{L}_{style}(\vec{a}, \vec{x}) Ltotal​(p​,a,x)=αLcontent​(p​,x)+βLstyle​(a,x)

根据这个总体差异，对 x⃗\vec{x}x 使用梯度下降法进行迭代更新，从而完成风格迁移。
